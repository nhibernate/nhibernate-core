Build 1.0.1.0
========================

- [NH-312] - Types from namespace System.Data.SqlTypes can now be used for
  nullable database columns. Usage is similar to Nullables, use
  type="NHibernate.UserTypes.SqlTypes.Sql*Type, NHibernate.UserTypes.SqlTypes".
- [NH-430] - Oracle BFILE user type. (Xianfeng Lu)
  Use type="NHibernate.UserTypes.Oracle.BFileType, NHibernate.UserTypes.Oracle"
  and see the documentation of the type.
- [NH-437] - MS Access Dialect and Driver (Lukas Krejci)
  To use, specify dialect as NHibernate.JetDriver.JetDialect and use OLEDB
  connection string (Jet dialect derives from OleDbDialect).

Build 0.99.2.0 (1.0-rc2)
========================
- NHibernate.Mapping.Attributes:
  - Can now be easily customized (breaking change).
  - Supports using any object in <discriminator-value>
    (and formatted enum)
  - Component can now be set through properties by using: ComponentPropertyAttribute.

Build 0.99.1.0 (1.0-rc1)
========================
- Merged documentation with NHibernate documentation.
- Added Parse to Nullable* types.
- Added a IUserType for strings to convert empty strings to NULLs in
  the database and vice versa. See the documentation for
  Nullables.NHibernate.EmptyStringType and EmptyAnsiStringType for details
  and usage information.
- NHibernate.Mapping.Attributes now throws an exception if a required attribute
  value was not set.
